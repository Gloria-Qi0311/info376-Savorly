Savorly: Recipe Search with BM25 and PyTerrier

Savorly is a simple recipe search system built in Python using PyTerrier and the BM25 ranking function. It takes a JSON recipe collection, performs text preprocessing (tokenization, stopword removal, stemming), builds a retrieval index, and allows users to run keyword queries from the command line.

The repository is structured as a small, self-contained information retrieval pipeline:

Data extraction and cleaning from recipes.json

Text preprocessing (stopword removal, stemming)

Index construction using PyTerrier

BM25 search over the preprocessed recipe corpus

2. Data Preprocessing (preprocess.py)

preprocess.py is responsible for transforming a raw recipe JSON file into a cleaned and indexed text corpus. 

preprocess

2.1 Input

A recipes.json file in the project root directory.

The JSON is expected to contain a top-level key such as "recipes" and 12 high-level recipe categories (e.g., "Lunch recipes", "Dinner recipes", etc.), each with nested subcategories and lists of recipes.

Each recipe record is used to construct a combined text field that includes:

Recipe name

Description

Ingredients

Steps

Preparation and cooking time

Difficulty level

This combined text is stored in a CSV file with two columns:

id — recipe identifier

text — concatenated textual description

The resulting CSV is written to output.csv. 

preprocess

2.2 Text Cleaning

preprocess.py uses:

nltk.corpus.stopwords (English stopwords)

PorterStemmer (Porter stemming)

Regular expressions to keep only alphabetic tokens of length ≥ 3

The cleaning function:

Lowercases the text

Extracts alphabetic tokens

Removes English stopwords

Applies stemming

Joins tokens back into a whitespace-separated string

The cleaned text is stored in a new column cleaned_text, and the full DataFrame is saved as:

cleaned_output.csv

assets/preprocessed_recipes_500.parquet (for downstream use) 

preprocess

2.3 Index Construction in preprocess.py

preprocess.py also constructs a PyTerrier index:

A corpus DataFrame is built with:

docno: string version of the original row index

text: the cleaned text (cleaned_text)

The corpus is converted to a list of dictionaries and passed to pt.IterDictIndexer

The index is written to assets/pp_index/ (creating the directory if necessary)

index_path = os.path.expanduser("./assets/pp_index")
os.makedirs(index_path, exist_ok=True)
indexer = pt.IterDictIndexer(index_path, overwrite=True)
index_ref = indexer.index(corpus.to_dict(orient="records"))


preprocess

3. BM25 Retrieval (BM25.py)

BM25.py focuses on loading the preprocessed dataset and running BM25 search. 

BM25

3.1 Data Loading

Reads assets/preprocessed_recipes_500.parquet using pandas.read_parquet

Rebuilds a corpus with:

docno: row index as string

text: cleaned recipe text (cleaned_text)

Builds a PyTerrier index at ./assets/pp_index (similar to preprocess.py)

3.2 Java and PyTerrier Initialization

BM25.py:

Uses /usr/libexec/java_home -v 17 to locate a Java 17 installation

Sets the following environment variables:

JAVA_HOME

JVM_PATH

PATH (prepended with the Java bin directory)

Initializes PyTerrier’s Java backend if not already started:

if not pt.java.started():
    pt.java.init()


This configuration is tailored for macOS. On other operating systems, the Java initialization may need to be adapted. 

BM25

3.3 BM25 Retrieval Setup

BM25.py constructs a BM25 retriever with explicit parameters:

bm25 = pt.BatchRetrieve(index_ref, wmodel="BM25",
                        controls={"bm25.b": 0.75, "bm25.k_1": 1.0})


These controls configure the BM25 b and k1 hyperparameters.

3.4 Interactive Search

The script then prompts for a query in the terminal:

search = input("Search: ")
bm25_results = bm25.search(search)
print(bm25_results)


Input: a free-form text query typed by the user

Output: a Pandas DataFrame printed to stdout, typically containing docno, score, and other retrieval metadata

5. Usage
5.1 Prepare the Data

Place a recipes.json file in the project root directory, matching the structure expected by preprocess.py (with a "recipes" object containing the 12 recipe categories and nested lists of recipes). 

preprocess

5.2 Run Preprocessing

From the project root:

python preprocess.py


This will:

Extract and format the raw recipe text

Generate:

output.csv

cleaned_output.csv

assets/preprocessed_recipes_500.parquet

Build a PyTerrier index in assets/pp_index/

5.3 Run BM25 Search

After preprocessing is complete, run:

python BM25.py


Then enter a query when prompted, for example:

Search: chicken pasta


You will see a DataFrame of ranked results printed to the terminal, with higher BM25 scores indicating more relevant documents.

7. Limitations and Possible Extensions

Current limitations (as implied by the code):

The system is command-line only; there is no web or GUI front-end.

The JSON input format is assumed and not validated extensively.

Retrieval uses pure BM25 without query expansion or semantic embeddings.

Possible extensions:

Add a web or GUI front-end for interactive search.

Integrate dense retrieval (e.g., sentence embeddings) and perform hybrid scoring (BM25 + embeddings).

Add support for filtering by category, difficulty, or time.

Improve error handling for malformed or missing fields in recipes.json.
